# Financial Ratios — 投后跟踪报告财务比率定义

This file is consumed by `analyst-deal:portfolio-tracking` via the `financial-analyzer` sub-agent. Treat the formulas below as **constants**. Do not invent or substitute formulas at runtime.

## Unit conversion

合并报表（人民币元）→ 投后报告呈现（万元）：
```
万元 = round(元 / 10000, 2)
```

Cross-check: 矽昌通信 2025-12-31 合并报表「短期借款 76,574,411.15 元」→ 投后报告「短期借款 7,657.44 万元」。系数 = 10000，舍入 2 位小数。**禁止使用其他系数或舍入策略。**

## Ratios (5 standard, all derived from 利润表)

| Name | Formula | Source lines (合并利润表) | Unit |
|---|---|---|---|
| 毛利率 | (营业总收入 − 主营业务成本) / 营业总收入 | 营业总收入；主营业务成本 | % |
| 销售费用率 | 销售费用 / 营业总收入 | 销售费用；营业总收入 | % |
| 管理费用率 | 管理费用 / 营业总收入 | 管理费用；营业总收入 | % |
| 研发费用率 | 研发费用 / 营业总收入 | 研发费用；营业总收入 | % |
| 财务费用率 | 财务费用 / 营业总收入 | 财务费用；营业总收入 | % |

**Computation rules:**

- 比率以百分比呈现，保留 1 位小数（例：`24.1%`，不是 `0.241` 也不是 `24.10%`）
- 营业总收入为零或缺失 → 该年所有比率显示 `—`，不引发除零
- 任一被除数缺失 → 当格显示 `—`
- **不允许扩展此列表** — 若将来需要资产负债率、流动比率等，先增补到本文件再用
- **不允许调用任何外部基准**（如行业平均、可比公司）— 报告中只呈现本公司多年纵向对比

## Year selection

- Agent 接受 1–4 个合并报表文件作为输入
- 报告期年份从合并报表表头识别（例 `2025年12月31日` → 列标 `20251231`）
- 输出表格列顺序：从早到晚（最左为最早年份，最右为最新年份）
- 缺失年份不补空列

## Audit guarantee

每一项数字在生成前必须可溯源到合并报表的具体行 + 行次号。如果某数字无法从输入合并报表中定位，**禁止输出该数字**，必须以 `—` 占位并在文字解读末尾的「数据缺口」段落明确标注。
