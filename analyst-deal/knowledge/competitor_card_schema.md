# Competitor Card Schema

`competitor-enricher` sub-agent 的输出契约。每家竞对返回一个**自包含、可直接嵌入主报告**的 markdown 块。主命令按用户指定顺序拼接 N 个档案块组成报告章节四「竞争对手情况」的主体。

**单 schema、按公司类型三分支**（决策见 `docs/adr/0001-competitor-card-single-schema-branched.md`）。先做类型分类，再按对应档位决定哪些节必填。

---

## 公司类型三档（先分类，再填卡）

| 档位 | 判定标准 | 财务表/芯片规格表/管理层表/客户与下游 | 股权结构 | 融资进展 | jina 预算 | 长度上限 |
|---|---|---|---|---|---|---|
| **listed 上市竞品** | 境内外任一证券交易所挂牌、有公开定期财报 | **必填** | 必填 | 不适用（写一行「上市公司，无私募轮次」；如 IPO 前有最后一轮可保留） | ≤16 | ≤1800 tokens |
| **pre-ipo 拟 IPO 竞品** | 未挂牌但有招股说明书申报稿/问询函 | 从招股书填到上市级；招股书未出则退回 non-listed | 必填 | 必填（按 non-listed 那套查私募轮次） | ≤12 | ≤1400 tokens |
| **non-listed 非上市创业竞品** | 无公开定期财报、也无招股书申报稿 | 可选；无数据时写 `不适用-未上市` 或 `数据缺口` | 必填 | **必填** | ≤8 | ≤800 tokens |

**分类规则：**

- 类型可由可选输入 `公司类型: listed/pre-ipo/non-listed/auto` 显式指定；缺省 `auto`。
- `auto` 时：enricher 在第一次股权 jina search 的返回里顺手判定（股票代码 / 「上市」「TWSE」「SH」「SZ」「HK」/ 「招股说明书」「IPO 辅导」「问询函」等信号），**不额外消耗 jina 调用**。
- 「上市」含境内外任一交易所（A 股 / 港股 / 台股 / 美股 ADR 等）。
- 三档互斥；判定不出时按 `non-listed` 走（最保守，预算最小）。
- 档案块标题行后**紧跟一行**类型标注：`> 公司类型：{{listed|pre-ipo|non-listed}}（{{判定依据，如 "TWSE:2379 台股上市"}}）`

## 字段填充硬规则

### 必须 (MUST)

1. 每个数字（持股比例、融资金额、估值、营收、毛利率、占比）**必须**附 URL 来源；无法溯源时写 `未公开` / `数据缺口`，**禁止**编造或基于行业惯例估计
2. 同一字段如果不同来源出现矛盾，**两个都列**并标来源，让分析师判断
3. 股权结构表至少覆盖前 5 大股东 + 我司可能存在的关联机构；前 5 之外可省略
4. Evidence 来源段必须给完整 URL，不允许只给域名；条数上限按公司类型档位
5. 「与本公司主产品的差异点」一句话**必须**呈现，否则 enricher 视为输出无效
6. **类型标注行**（`> 公司类型：...`）必须紧跟标题；listed/pre-ipo 必须产出财务表、芯片规格表、客户与下游、管理层四节（无数据的单元格写 `未披露`，但节不可整体省略）
7. 财务表币种**必须**为报告原币种并明标单位；**禁止**做汇率换算
8. "三年一期"缺某期时**必须**写 `截至 {{YYYY-MM}} 未披露` 并用最近可得期，不得留空或编造

### 不允许 (MUST NOT)

1. 不允许在档案块中写主观判断（如"该竞品威胁较大"）— 主观判断由主命令在「五、小结」中统一处理
2. 不允许混入本公司的信息 — 档案块只描述被研究的竞对
3. 不允许出现"近期""目前"等模糊时间词 — 必须给到 `YYYY-MM` 精度
4. 不允许使用"约""估计""大概" — 数字给精确值或写 `未公开`

### 财务文字解读边界（与 MUST NOT #1 配套）

财务表后如附文字解读，仅允许**纯描述性趋势句**，禁因果归因与评价形容词：

- ✅ 允许：数值、YoY%、`升 / 降 / 持平` + 期间。例：「整体毛利率 2023→2025 由 42.0% 降至 38.0%；同赛道段营收占比三年由 31% 升至 37%。」
- ❌ 禁用词：承压 / 恶化 / 强劲 / 优异 / 亮眼 / 疲软 / 反映竞争加剧 / 盈利能力下滑 / 表明 / 由于…导致（任何因果归因或评价形容词）
