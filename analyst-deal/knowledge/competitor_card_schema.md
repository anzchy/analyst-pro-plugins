# Competitor Card Schema

`competitor-enricher` sub-agent 的输出契约。每家竞对返回一个**自包含、可直接嵌入主报告**的 markdown 块。主命令按用户指定顺序拼接 N 个档案块组成报告章节四「竞争对手情况」的主体。

---

```markdown

|---|---|---|---|---|
| 1 | {{股东名}} | {{xx.xxxx%}} | {{YYYY-MM-DD}} | {{机构 / 关联企业}} |
| ... | ... | ... | ... | ... |

> 数据来源：{{aiqicha.baidu.com 或 工商系统具体 URL}}；查询日期：{{YYYY-MM-DD}}

**产品方向：**

{{1-2 段，~150-250 字。要求：
```

## 字段填充硬规则

### 必须 (MUST)

1. 每个数字（持股比例、融资金额、估值）**必须**附 URL 来源；无法溯源时写 `未公开` / `数据缺口`，**禁止**编造或基于行业惯例估计
2. 同一字段如果不同来源出现矛盾（如 36kr 说 1 亿、官网新闻说 1.2 亿），**两个都列**并标来源，让分析师判断
3. 股权结构表至少覆盖前 5 大股东 + 我司可能存在的关联机构；前 5 之外可省略
4. Evidence 来源段必须给完整 URL，不允许只给域名
5. 「与本公司主产品的差异点」一句话**必须**呈现，否则 enricher 视为输出无效

### 不允许 (MUST NOT)

1. 不允许在档案块中写主观判断（如"该竞品威胁较大"）— 主观判断由主命令在「五、小结」中统一处理
2. 不允许混入本公司的信息 — 档案块只描述被研究的竞对
3. 不允许出现"近期""目前"等模糊时间词 — 必须给到 `YYYY-MM` 精度
4. 不允许使用"约""估计""大概" — 数字给精确值或写 `未公开`
