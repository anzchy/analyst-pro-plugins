# 情报评分规则 (Triage Rules)

> market-intel subagent 对每条新闻/信息进行评分的标准。

## 五级评分标准

| 分数 | 级别 | 定义 | 处理方式 |
|---|---|---|---|
| **5** | 紧急/重大 | 直接影响在投项目或活跃 DD 的信息 | **立即推送 Telegram** |
| **4** | 重要 | 重点赛道的重要变化（大额融资、政策变动、技术突破） | 写入 daily_brief，早报发送 |
| **3** | 关注 | 赛道相关有价值信息，但不紧急 | 写入 daily_brief |
| **2** | 背景 | 一般行业动态，有参考价值 | 仅归档 news_archive |
| **1** | 噪音 | 与投资决策无关的信息 | 仅归档或忽略 |

## 评分维度

### 与在投项目的相关性 (权重最高)
- 涉及被投企业的直接新闻 → +2
- 涉及被投企业竞品的重要动态 → +1
- 涉及活跃 DD 项目的赛道 → +1

### 时效性
- 今日发生 → +1
- 本周发生 → +0
- 历史信息 → -1

### 决策影响力
- 可能改变投资判断 → +2
- 补充既有认知 → +0
- 纯知识性无决策价值 → -1

### 信息独特性
- 独家/首发信息 → +1
- 多家已报道的重复信息 → -1

## 自动 Score-5 触发条件

以下情况不论评分直接推送 Telegram：
- 被投企业出现负面新闻（裁员、高管离职、诉讼）
- 重点关注赛道出现重大政策变动
- 竞品机构对我们重点赛道做出大额投资
- BIS Entity List 更新涉及投资关联方

## 输出格式

每条情报：

```markdown
**[Score: X] [赛道] 标题**
来源: [媒体名] | 日期: YYYY-MM-DD
摘要: [1-2 句话]
与我们的关联: [为什么重要]
```

_评分标准根据投资策略变化持续调整。_
