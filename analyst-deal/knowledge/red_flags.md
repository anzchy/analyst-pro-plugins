# 投资风险红旗清单 (Red Flags Checklist)

> deal-analyst 在 Phase 2 深度分析时逐项核查。
> 任何 RED FLAG 触发 → 必须在报告中显著标注。

---

## 团队红旗

- [ ] 创始人有过失败退出且未坦诚披露
- [ ] CTO 仅有学术背景，无产品化/量产经验
- [ ] 核心团队近期有离职（尤其技术负责人）
- [ ] 创始人同时运营多个公司
- [ ] 股权结构异常（创始人占比过低、代持、多层嵌套）
- [ ] 团队简历夸大（声称"前 Google"但实为外包）

## 财务红旗

- [ ] 收入增长依赖单一大客户（> 50%）
- [ ] 关联交易收入占比高
- [ ] 现金消耗速度远超融资节奏（Runway < 6 个月仍不启动新一轮）
- [ ] 财务数据前后不一致（BP vs 尽调数据不匹配）
- [ ] 估值严重偏离同行可比（无合理解释）

## 技术红旗

- [ ] 核心技术主张无同行评审论文或专利支撑
- [ ] 技术指标声称远超行业基准（偏差 > 20%）
- [ ] 产品 demo 仅为模拟/渲染，无实物原型
- [ ] 依赖单一供应商/代工厂且无备选
- [ ] 无法提供第三方测试/认证数据

## 市场红旗

- [ ] TAM 估算引用不可靠来源或明显膨胀
- [ ] 商业化时间线不切实际（硬科技尤其注意）
- [ ] 已有巨头进入且公司无差异化壁垒
- [ ] 客户声称"有意向"但无 LOI/MOU/订单
- [ ] 市场天花板太低（SAM < ¥10 亿且增速平缓）

## 合规红旗

- [ ] 涉及出口管制受限技术（见 `export_control_rules.md`）
- [ ] 公司或创始人在 BIS Entity List
- [ ] 军民两用技术未做合规评估
- [ ] 数据安全/隐私合规未就绪
- [ ] 行业监管许可未取得（如核安全审批、航天许可）

## 交易结构红旗

- [ ] 对赌条款过于苛刻
- [ ] 优先清算权层级过多
- [ ] 反稀释条款异常（full ratchet）
- [ ] 创始人有大额套现（secondary）
- [ ] 竞业限制缺失或过松

---

## 严重程度分级

| 级别 | 定义 | 处理 |
|---|---|---|
| **RED FLAG** | 可能导致投资失败的根本性风险 | 必须在报告中醒目标注，IC 会前必须讨论 |
| **YELLOW FLAG** | 需要关注但可通过条款/尽调化解 | 在报告中列出，建议跟进 |
| **GREEN** | 已核查无异常 | 简要标注通过 |

---

## 硬科技专项红旗 (Hard Tech Kill Criteria)

> Phase 1 硬停止检查（kill flags）位于 `report_template.md` 第八节，是筛选阶段自包含的快速检查。
> 本节为 Phase 2 深度分析提供更全面的硬科技特异性红旗参考。

- [ ] TRL < 3 却声称"量产就绪"（TRL 与声明严重不匹配）
- [ ] 创始人声称"量产"但无晶圆代工/制造合作伙伴
- [ ] 专利组合授权 3 年以上但零被引用（专利质量存疑）
- [ ] 团队全部为学术背景，无产业 COO 且无引入计划
- [ ] 声称国产替代但产品不对应任何"卡脖子"品类
- [ ] 同 TRL 阶段可比公司估值的 3 倍以上溢价（无合理解释）
- [ ] TAM < 50 亿元且无扩展路径（Phase 1 粗筛；本文件 §市场红旗 的 SAM < 10 亿 为 Phase 2 深度检查）
